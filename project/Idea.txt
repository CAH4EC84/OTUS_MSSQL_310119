--Описание проекта
1) Данные для отчетов поступаю раз в сутки из OLTP БД
	Тут все понятно
2) Для генерации отчетов испульзуется исключительно средства MS SQL
	Тут полет фантазии - решения в инете не подсматривались.
	2.1) Есть список пользователь с определенной иерархией папок (\\somePath\Julia ; \\somePath\Sergey etc)
		Есть базовая папка в который хранятся шаблоны отчетов (XML файлы) с описанием \ примерами сгенерированных отчетов.
	2.2) Запрос на генерацию отчета от пользователя поступает в XML файле 
		Список отчетов предустановлен, расширяем по мере разработки. Генерация через хранимые процедуры \ функции.
		Пользователель копирует из базовой папки необходимый ему шаблон отчета, устанавливает требуемые параметры.
	2.3) Планировщик заданий проверяет пользовательские папки по таймеру, если появилось новое задание :
		Регистрирует его в таблице Reports.
		Запускает генерацию выбранного отчета.
		Выгружает софрмированный отчет.
		Делает пометку о готовности.
		Опционально уведомляет пользователя о готовности отчета. https://habr.com/ru/post/179819/
	2.3) Сформированный отчет выгружается в директорию пользователя (.\compliteReports).
		Тут какое-то правило именования файлов.
	2.4) Ведется таблица кому/когда/по какому XML / и т.д был сформирован отчет
